# Навыки и Алгоритмы

## Работа с Git и GitHub

### Настройка безопасного пуша (Гарантированный алгоритм)
1. **Получение токена**: Токен должен иметь права `repo` и `workflow`.
2. **Настройка учетных данных**:
   - Использовать `git config --global credential.helper store`.
   - Сохранить токен в файле `~/.git-credentials` в формате: `https://<username>:<token>@github.com`.
   - Не указывать токен напрямую в URL репозитория.
3. **Работа с ветками**:
   - Создавать новые ветки для каждой задачи/изменения.
   - Выполнять коммиты с понятными сообщениями.
   - Отправлять ветку в удаленный репозиторий через `git push -u origin <branch-name>`.
4. **Гарантия отправки (Важно!)**:
   - После выполнения `git push` **обязательно проверить код возврата (exit code)**.
   - Если код возврата `0` — операция успешна, ветка на сервере.
   - Если код возврата не `0` — прочитать вывод ошибки, исправить причину (например, истек токен) и повторить.
   - Не считать операцию успешной без явного подтверждения от терминала или сервера.

### Создание Pull Request (Merge Request)
   - После успешного пуша новой ветки необходимо создать запрос на слияние (Pull Request) в основную ветку (`main`).
   - Это можно сделать через веб-интерфейс GitHub или используя CLI: `gh pr create --title "..." --body "..." --base main`.
   - Шаг является обязательным для завершения цикла разработки и попадания кода в продакшн.

### Пример команды для настройки
```bash
git config --global credential.helper store
echo "https://<username>:<token>@github.com" > ~/.git-credentials
git config --global user.email "<email>"
git config --global user.name "<name>"
```
